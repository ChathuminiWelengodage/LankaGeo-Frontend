# Map Legends & Endpoint Changes — Frontend Handover **File:** `MAP_LEGENDS_AND_ENDPOINT_CHANGES.md`  **Date:** 2026-06-13  **Audience:** Frontend Team--## Executive Summary Backend endpoints for map visualizations were standardized and extended for two separate map flows:- Live Flood Analysis: `POST /api/v1/analyze/ live`  - Historical Trend Analysis: `POST /api/v1/ analyze/trend` Responses are mocked in tests and now include consistent tile URL formats, geojson polygons with severity properties, and metadata needed by the frontend. Map legend color mappings and layer identifiers were clarified to avoid confusion between live and historical layers.--## What Changed (Backend) 1. Tile URL naming conventions- Live: contains the segment `live` and a map id pattern like `.../maps/live-flood{lat}-{lng}/tiles/{z}/{x}/{y}`   - Historical composite: `.../maps/ historical-composite-{mapId}/tiles/{z}/{x}/{y} `   - Historical composite: `.../maps/ historical-composite-{mapId}/tiles/{z}/{x}/{y} `   - Historical heatmap: `.../maps/historicalheatmap-{mapId}/tiles/{z}/{x}/{y}` 2. GeoJSON & polygons- GeoJSON FeatureCollection returned for live analysis.- Each Feature has `properties.severity_level` with integer values `1`, `2`, `3`.- Polygons are validated to be closed and have >= 3 unique points. 3. Metadata fields added/guaranteed- `affected_area_km2` (float)- `confidence_score` (0-100 float)- `satellite_source` (string)- `cloud_cover_pct` (0-100 float)- `risk_level` (string: "Low"|"Moderate"|"Critical")- `analysis_timestamp` (ISO8601) 4. Historical response additions- `years_data`: array of `{ year, value, details }` where `value` is 0-100- `composite_tile_url`, `trend_heatmap_url`   - `zone_count_by_severity`: list of severity counts 5. Tile layers are guaranteed to be distinct strings so frontend can map them to different layer IDs.--## Map Legend & Color MappingLive Flood severity numeric mapping:- `1` → Blue `#0000FF` (Seasonal/Water) — label: `Low`- `2` → Amber `#FFA500` (Moderate Flood) — label: `Moderate`- `3` → Red `#FF0000` (Critical Flood) — label: `Critical` Historical Zone severity labels:- `Low` → Blue palette- `Medium` → Yellow/Amber palette- `High` → Red palette Legend requirements:- Show color swatch, label, example polygon opacity, and sample confidence range.- Clearly annotate which layer the legend refers to (Live vs Historical).--## Frontend Change Plan (concise) Goal: Implement two separate map flows (Live and Historical) with clear legends and metadata panels. 1.
